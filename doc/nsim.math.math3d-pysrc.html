  35  """Homogeneous Transformation Matrices and Quaternions. 
  36   
  37  A library for calculating 4x4 matrices for translating, rotating, reflecting, 
  38  scaling, shearing, projecting, orthogonalizing, and superimposing arrays of 
  39  3D homogeneous coordinates as well as for converting between rotation matrices, 
  40  Euler angles, and quaternions. Also includes an Arcball control object and 
  41  functions to decompose transformation matrices. 
  42   
  43  :Authors: 
  44      `Christoph Gohlke <http://www.lfd.uci.edu/~gohlke/>`__, 
  45      Laboratory for Fluorescence Dynamics, University of California, Irvine 
  46   
  47  :Version: 
  48      20091230 
  49   
  50  Requirements 
  51  ------------ 
  52   
  53  * `Python 2.6 <http://www.python.org>`__ 
  54  * `Numpy 1.4 <http://numpy.scipy.org>`__ 
  55  * `transformations.c 20091230 <http://www.lfd.uci.edu/~gohlke/>`__ 
  56    (optional implementation of some functions in C) 
  57   
  58  Notes 
  59  ----- 
  60   
  61  The API is not stable yet and is expected to change between revisions. 
  62   
  63  This Python code is not optimized for speed. Refer to the transformations.c 
  64  module for a faster implementation of some functions. 
  65   
  66  Documentation in HTML format can be generated with epydoc. 
  67   
  68  Matrices (M) can be inverted using numpy.linalg.inv(M), concatenated using 
  69  numpy.dot(M0, M1), or used to transform homogeneous coordinates (v) using 
  70  numpy.dot(M, v) for shape (4, \*) "point of arrays", respectively 
  71  numpy.dot(v, M.T) for shape (\*, 4) "array of points". 
  72   
  73  Use the transpose of transformation matrices for OpenGL glMultMatrixd(). 
  74   
  75  Calculations are carried out with numpy.float64 precision. 
  76   
  77  Vector, point, quaternion, and matrix function arguments are expected to be 
  78  "array like", i.e. tuple, list, or numpy arrays. 
  79   
  80  Return types are numpy arrays unless specified otherwise. 
  81   
  82  Angles are in radians unless specified otherwise. 
  83   
  84  Quaternions w+ix+jy+kz are represented as [w, x, y, z]. 
  85   
  86  A triple of Euler angles can be applied/interpreted in 24 ways, which can 
  87  be specified using a 4 character string or encoded 4-tuple: 
  88   
  89    *Axes 4-string*: e.g. 'sxyz' or 'ryxy' 
  90   
  91    - first character : rotations are applied to 's'tatic or 'r'otating frame 
  92    - remaining characters : successive rotation axis 'x', 'y', or 'z' 
  93   
  94    *Axes 4-tuple*: e.g. (0, 0, 0, 0) or (1, 1, 1, 1) 
  95   
  96    - inner axis: code of axis ('x':0, 'y':1, 'z':2) of rightmost matrix. 
  97    - parity : even (0) if inner axis 'x' is followed by 'y', 'y' is followed 
  98      by 'z', or 'z' is followed by 'x'. Otherwise odd (1). 
  99    - repetition : first and last axis are same (1) or different (0). 
 100    - frame : rotations are applied to static (0) or rotating (1) frame.
